Doświadczenie projektowe

  • Aplikacja "Book reading" Projekt na Github

    Technologii: [Javascript, Axios, React, React Router, React Responsive, React Redux, Styled Components, Formik, Yup, GIT, Netlify, Heroku]

    Moja rola w projekcie: team lead

    To jest wspólny studenckie projekt Kursa full stack. Proponujemy responsywną aplikacje dla kontrola ilości przeczytanych książek. W app wdrożona rejestracja i logowanie (obejmuje z użyciem Google profile).Aplikacja ma 2 trasy: biblioteka i trening. Biblioteka pokaż swoją listę książek ze statusem książkę (już przeczytane, czytamy teraz, będzie przeczytane). Trening może pokaż statystyki, licznik czasu do końca traningą i inna informacja przez twój aktualne szkolenie czytelnicze. Front-end pracuje z Netlify, Back-end pracuje z Heroku.

  • Aplikacja "My contacts" Projekt na Github

    Technologii: [Javascript, Axios, React, React Router, RTK Query, Material-UI, GIT]

    Prosty React app z routingu do przechowywania kontaktów. Na przykład, twój komórkowy list. Aplikacja ma proste rejestracją I pracuje z backend na żywo. Ja użył niektórych przygotowanych stylizowane komponenty z biblioteki Material-UI i uświadomić sobie zmieniające się motywy (jasne albo temne) dla liepiego usability.

  • Aplikacja "Movie finder" Projekt na Github

    Technologii: [Javascript, Axios, React, React Router, Emotion]

    Ta React aplikacja jest przydatny do wyszukiwania informacji o niektórych filmach. W jakości backend ja używam themoviedb.org API. To jest użycie async załadunek JS-kodu dla każdą trasę w aplikacji. Ja używam routing w aplikacje dla malejące ładowanie danych i przyspieszenie pracy aplikacji.
